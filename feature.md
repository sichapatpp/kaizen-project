# ข้อจำกัดของระบบและสิ่งที่ควรพัฒนาต่อยอดในอนาคต (System Limitations & Future Enhancements)

## 📌 ข้อจำกัดของระบบในปัจจุบัน (Current System Limitations)

1. **การจัดการ Role และสิทธิ์การใช้งาน**
   - สิทธิ์บางอย่างยังถูกฝังอยู่ในระดับ Code (Hard-coded ใน Route Middleware เช่น `role:manager,chairman,admin`) หากองค์กรมีหน่วยงานหรือระดับขั้นการอนุมัติเพิ่มขึ้น จะต้องให้นักพัฒนาเข้ามาแก้ไขโค้ด
   - การจัดการสิทธิ์ย่อย (Permissions) ของแต่ละ Role ยังอาจต้องพึ่งตรรกะแบบตายตัว ขาดระบบหน้าจอให้ Admin เข้าไปควบคุมสิทธิ์ย่อยได้อย่างอิสระ (Dynamic Permissions)

2. **การทำงานร่วมกันโดยหลายคนพร้อมกัน (Collaboration Constraint)**
   - แม้ระบบจะสามารถเพิ่ม "ผู้ร่วมกิจกรรม" (Participants) ได้ แต่เมื่อถึงขั้นตอนการแก้ไขแบบฟอร์ม (Form Editing) ยังไม่รองรับให้หลายๆ คนช่วยพิมพ์และแก้ไขไปพร้อมกัน (Real-time Collaborative Editing) ผู้ใดกด Save ทีหลังสุดข้อมูลจะทับคนก่อนหน้า

3. **ระบบบริหารจัดการไฟล์ (File Management)**
   - รูปภาพถูกอัปโหลดและเก็บไว้ในเซิร์ฟเวอร์ (Local File Storage) ซึ่งหากใช้งานในระยะยาว มีผู้ใช้จำนวนมากอัปโหลดไฟล์ขนาดใหญ่ อาจทำให้พื้นที่ดิสก์เต็มเร็ว และยังไม่มีการบีบอัดรูปให้อัตโนมัติ (Auto-Compression/Resizing) เพื่อประหยัดพื้นที่
   - หากมีการขยายสเกลระบบ อาจจำเป็นต้องเปลี่ยนไปใช้ Cloud Storage แบบ AWS S3

4. **ระบบ Queue และการประมวลผลพื้นหลัง**
   - ระบบอีเมลใช้ Queue ซึ่งเป็นการทำงานเบื้องหลังแล้ว แต่ในเซิร์ฟเวอร์จริงก็มีความจำเป็นต้องตั้งค่า Queue Worker / Supervisor ไว้ให้ระบบทำงานได้อย่างต่อเนื่อง หากโฮสติ้งไม่รองรับ Supervisor อาจทำให้ขั้นตอนค้างได้

## 🚀 สิ่งที่ควรพัฒนาต่อยอดในอนาคต (Future Enhancements)

1. **Notifications ผ่านทางเลือกอื่น (LINE Notify / MS Teams)**
   - แจ้งเตือนสถานะต่างๆ (เช่น มีคำขออนุมัติใหม่เข้า, โปรเจกต์ได้รับการอนุมัติ) โดยยิงเข้าหา LINE Notify, Microsoft Teams, หรือ Slack ของผู้ใช้งาน เพื่อความรวดเร็วในการติดตามงานแทนการพึ่งพาเพียง Email

2. **ระบบ Dynamic Approval Workflow**
   - สร้างฟีเจอร์สำหรับกำหนดขั้นตอนการอนุมัติ (Approval Configurations) ผ่านหน้าบ้าน โดยให้ Admin ลากวาง (Drag-and-Drop) ผู้ที่มีอำนาจอนุมัติ เพื่อความยืดหยุ่นหากโครงสร้างบริษัทเปลี่ยน ไม่ต้องแกะโค้ด

3. **ปรับปรุงส่วนของ Dashboard และ Business Intelligence (BI)**
   - เพิ่มระบบการฟิลเตอร์ข้อมูลเชิงลึกข้ามตัวแปร เช่น เปรียบเทียบผลงานแต่ละแผนก (Department / Division) ในรายไตรมาส
   - เพิ่มคาดการณ์ (Forecasting / Trending) ช่วยให้ผู้บริหารเห็นแนวโน้มการลดงบประมาณ หรือเพิ่มรายได้ล่วงหน้าได้

4. **ระบบ Gamification และ Leaderboard**
   - เพิ่มระบบ Level หรือ Badges ให้กับพนักงานที่อนุมัติกิจกรรม Kaizen จนสำเร็จ 
   - มีระบบ Leaderboard ของแต่ละไตรมาส เพื่อแสดงชื่นชมและกระตุ้นการมีส่วนร่วมกับนวัตกรรมใหม่ๆ ของคนในองค์กร

5. **ระบบ Progressive Web App (PWA)**
   - ทำให้ Web Application นี้สามารถติดตั้งเป็นชอร์ตคัทลงในมือถือของผู้ใช้งานได้ (Installable) 
   - อาจเพิ่มการรองรับ Offline-first เล็กๆ น้อยๆ โดยการบันทึก Draft ลง LocalStorage แล้วบังคับ Sync เมื่อเชื่อมต่ออินเทอร์เน็ต

6. **ระบบ Import/Export ด้วย Excel**
   - พัฒนาระบบให้สามารถ Import ผู้จัดทำกิจกรรมต่างๆ ทีละมากๆ โดยอัปโหลดไฟล์ Excel ได้เลย โดยไม่ต้องไล่กรอกทีละคน
   - อัปเกรดการปรับ Layout และ Template ของการ Export PDF ให้ยืดหยุ่นกว่าเดิม สามารถเปลี่ยน Header Footer ขององค์กรจากหน้าตั้งค่าได้เลย
